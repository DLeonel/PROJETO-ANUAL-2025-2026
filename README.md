## Branch `develop`

O branch `develop` possui como principal finalidade servir como ambiente intermediário de integração e revisão das funcionalidades desenvolvidas pelos integrantes do projeto.

Todas as partes concluídas ou parcialmente finalizadas devem ser enviadas inicialmente para o `develop`, onde serão realizadas:

- revisões estruturais;
- análise de compatibilidade;
- verificação de organização;
- padronização do código;
- remoção de arquivos desnecessários;
- avaliação de conflitos;
- testes de funcionamento;
- validação geral das implementações.

Após a revisão e validação coletiva, as funcionalidades aprovadas serão integradas ao branch principal (`main`) através do processo de mesclagem.
