# NEXATEACH — Projeto Anual 2025–2026

## Sobre o Projeto

O presente projeto foi desenvolvido no âmbito do Projeto Anual Escolar 2025–2026 realizad, como parte da avaliação final prática e técnica dos integrantes do grupo.

## Base de Conhecimento e Desenvolvimento

O presente projeto está sendo desenvolvido com base nos conhecimentos adquiridos ao longo da formação académica, complementados por pesquisas independentes, estudos individuais e recursos externos de aprendizagem.

Grande parte do conhecimento aplicado durante o desenvolvimento foi obtida através de:

- pesquisas técnicas;
- documentação oficial;
- vídeo-aulas;
- livros;
- PDFs;
- cursos complementares;
- práticas individuais;
- compartilhamento de conhecimento entre os integrantes do grupo.


# Tema do Projeto

O presente projeto consiste no desenvolvimento de um:

## Website Corporativo com Suporte Mobile

O sistema está a ser construído com base em uma empresa fictícia do setor de ITT (Informática, Telecomunicações e Tecnologias), criada pelo grupo com inspiração em empresas reais presentes no mercado angolano.

A empresa fictícia criada chama-se:

# NEXATEACH

A Nexateach foi concebida com características semelhantes a empresas de telecomunicações e tecnologia presentes no mercado angolano, seguindo uma estrutura corporativa moderna e adaptada à realidade do setor.

O objetivo é simular um ambiente empresarial real, desenvolvendo uma solução moderna, corporativa, responsiva e estruturada de acordo com padrões atuais do mercado tecnológico.

---

# Objetivo Principal

O projeto visa desenvolver:

- um website corporativo moderno;
- uma interface profissional e responsiva;
- estrutura organizacional empresarial;
- sistema visual alinhado ao setor tecnológico;
- integração futura com aplicação mobile;
- base estrutural para expansão do sistema futuramente.

---

# Desenvolvimento do Projeto

O desenvolvimento do sistema está dividido em etapas e realizado de forma colaborativa através de branches individuais.

A estrutura do desenvolvimento foi organizada para permitir:

- maior organização;
- redução de conflitos;
- desenvolvimento paralelo;
- melhor controle de versão;
- revisão coletiva antes da integração final.

As tarefas e funcionalidades estão a ser distribuídas de acordo com as necessidades do projeto e com as áreas de atuação definidas durante o desenvolvimento.
